Manual Testing Documentation

Project Overview

The WEare social network platform was tested manually at http://localhost:8081. The objective was to validate its core functionalities against the specified requirements, identify bugs, and assess missing features.

Scope

Manual testing focused on validating the following:

  • Functional requirements for public, private, and administrative features.
  • System compliance with user interaction standards (registration, login, post creation, etc.).
  • Identification of bugs, missing functionalities, and deviations from requirements.

Functionalities Tested

1. Functionalities Covered

  • Registration and Login: Validated field constraints, including username uniqueness, password strength, and email format. Tested anonymous user registration and login processes.
  • User Profile Management: Verified updates to name, profile picture, visibility settings, and optional fields like age and nationality.
  • Post Creation and Management: Tested creating posts with public and connection-only visibility.
  • Connections: Validated connection requests, approvals, and disconnections.
  • Public Feed: Ensured that public posts were displayed chronologically. Verified profile search using name and email.
  • Administrative Privileges: Tested editing and deleting users, posts, and comments by admin users.

Findings

1. Bugs Identified

  • IAG-64: Registration allowed with an already registered email.
  • IAG-66: Registration allowed with a password shorter than 8 characters.
  • IAG-67: Registration allowed with a password missing uppercase letters or special characters.
  • IAG-63: Profile search by email failed to display registered users.
  • IAG-69: Profile search by UserID failed to display existing profiles.
  • IAG-68: Post title disappeared after changing visibility to "public."
  • IAG-70: Private posts of non-connected users were visible to others.
  • IAG-79: Email verification bypass allowed login without confirmation.
  • IAG-74: System failed to retrieve profiles in search due to database errors.
  • IAG-76: Missing functionality for changing the password in the account settings.
  • IAG-78: Missing identity verification functionality for users.
  • IAG-104: All user profiles displayed when anonymous users accessed the search.

2. Missing Functionalities

  • Email Verification: Registration did not include email confirmation.
  • Identity Verification: No feature to verify user identity with ID and selfie.
  • Post Interactions: Users were unable to like or comment on posts.
  • Administrative Tools: Limited admin control over user connections and privacy settings.

Risk Management

  • Delays in Environment Setup: Addressed through early configuration and daily monitoring.
  • Requirement Changes: Managed by isolating features into specific sprints.
  • Dependency on External Systems: Mitigated through backup environments and manual workarounds.

Test Coverage and Gaps

Coverage

  • Registration and login functionality.
  • Basic profile and post management.
  • Core public and private features.

Gaps

  • Advanced user verification mechanisms.
  • Interactions on posts (comments, likes).
  • Enhanced admin tools for moderating connections and privacy settings.

Conclusion

Findings Summary

  • Bugs: 12 critical and functional bugs identified, affecting core workflows like registration, login, and profile search.
  • Missing Features: Essential functionalities such as email verification and post interactions are absent.
  • Requirements Compliance: The platform does not fully meet the functional requirements.

Recommendation

  • Fixing critical bugs, especially those related to registration, login, and profile visibility.
  • Implementing missing features, including email and identity verification.
  • Enhancing post management and user interaction functionalities.
