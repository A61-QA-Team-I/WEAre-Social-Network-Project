Test Reporting for WEAre Social Network Application

Testing Process Overview

The testing results of the WEAre Social Network application show that, despite covering the core functionalities, the application is not ready for client use. Some of the core features, such as registration and login, do not function according to the requirements, revealing serious issues with security and user authentication.

The application does not enforce proper password requirements – there is no minimum length or complexity, putting accounts at risk. Furthermore, the system allows users to register and log in with the same email addresses, which violates account uniqueness.

The user search functionality is not working correctly and lacks basic functionality to search for users by name, which complicates navigation within the application.

Additionally, the API has issues with incomplete documentation and missing endpoints, which hinder integration and usability.

The most serious issue is the lack of protections against unauthorized privilege escalation – regular users can become administrators, which exposes the application to significant abuse risks.

Other problems include incorrect post visibility, where posts remain public even when they shouldn’t.

These issues highlight the need for significant improvements in the security, functionality, and stability of the application.

The application is not ready for production and must undergo major fixes before it can be released to end users.

Scope of Testing and Activities Performed

  • Manual Testing: Covered main scenarios like registration, login, post creation, likes, comments, and user profile management.
  • API Testing with Postman: Validated CRUD operations for users and posts, authentication, and other key functions.
  • Automation with Selenium WebDriver: Tested critical scenarios like registration, login, and post creation.
  • Compatibility Testing: Conducted on Google Chrome, Mozilla Firefox, and Safari on macOS and Windows.
  • Performance Testing: Verified load resilience and response times using JMeter.
  • Security Testing: Checked essential data protection requirements and access controls.

Conclusions and Risk Assessment

The overall quality of the application is promising, but it still requires significant improvements. The risks associated with moving to production are significant at this stage. Additional work is needed on the application's stability and security to ensure that it will be safe and reliable for end users. The application is not ready for production release until these issues are resolved and the necessary stability and protection against potential vulnerabilities are ensured.

Recommendations

  • Fix the issues with password requirements by enforcing a minimum length and complexity to ensure account security.
  • Resolve the issue with user registration and login allowing duplicate email addresses to ensure account uniqueness.
  • Restore the user search functionality by adding the ability to search for users by name, to simplify navigation within the application.
  • Improve the API documentation by adding missing endpoints and enhancing the overall description to ease integration and usage.
  • Implement protections against unauthorized privilege escalation to prevent regular users from becoming administrators.
  • Correct the issue with private profile settings so that users who have set their profiles to private can ensure that only the information they want to be visible is accessible to others.
